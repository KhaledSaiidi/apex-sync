I built a production-like Kubernetes platform on a local kind cluster by replacing the default networking with Cilium (eBPF), exposing services externally using MetalLB to assign a real LoadBalancer IP from the Docker network, and implementing modern ingress using the Gateway API with Istio as the Gateway controller, which dynamically provisions an Envoy-based gateway to route traffic. You deployed an application served via a Service + HTTPRoute, mapped a real domain (app.kube-forge.com) through Route53 DNS to the MetalLB IP, and automated TLS using cert-manager with Let’s Encrypt (DNS-01 via Route53), enabling certificate issuance (including wildcard capability) and secure HTTPS termination at the Gateway. On top of that, you enforced platform-level behavior using Kyverno, automatically labeling the default namespace for Istio sidecar injection and triggering deployment restarts to ensure consistency. The result is a fully integrated system where external traffic flows from DNS → LoadBalancer IP → Istio Gateway → HTTPRoute → Service → Pod, with automated security, networking, and policy layers behaving like a real cloud-native platform.
